# ゲーム外部・内部仕様書：宇宙船プラットフォームランナー（仮題）

本ドキュメントは、スマートフォン向けWebブラウザで動作する、PixiJSを使用した2D横スクロール・プラットフォームゲーム（無限ランナー）の設計・開発用仕様書である。アセット（画像素材）を使用せず、PixiJSの描画機能（`PIXI.Graphics`）とパーティクル演出のみでスタイリッシュに完結させる実装を想定している。

---

## 1. 全体概要

### 1.1. ゲームコンセプト
* **コア体験:** 「慣性」と「制限されたジェット噴射」による、スリリングかつ精密な宇宙船コントロール。
* **ジャンル:** 横スクロール・プラットフォーム・アクション（無限ランナー）
* **ターゲット:** 短時間でリプレイ性を求めるカジュアルゲーマー、操作の熟練を楽しめるプレイヤー。
* **対応環境:** スマートフォン（Webブラウザ / モバイルWebView）※タッチ・ドラッグ操作最適化。
* **主要技術:** PixiJS (HTML5 / JavaScript または TypeScript)

### 1.2. 基本ルール
* プレイヤーは宇宙船を操作し、ランダムに生成される宙に浮いた足場（浮島）を渡り歩きながら、右方向へどこまで遠くへ進めるかを競う。
* 画面右へ進んだ距離がそのまま「スコア」となる。
* **ゲームオーバー条件:** 宇宙船が足場に着地できず、画面最下部（奈落）へ落下した瞬間に即ゲームオーバーとなる。

---

## 2. 画面遷移とゲーム状態（ステート）管理

ゲームは3つのメイン画面（シーン）で構成され、それぞれの状態を管理する。

```plaintext
[ ① タイトル画面 ] --(STARTタップ)--> [ ② ゲームプレイ画面 ]
^                                      |
|                               (画面最下部へ落下)
|                                      v
(TITLEタップ) ----------------------- [ ③ スコア表示画面 ]
|                                      |
+-----------(RETRYタップ)--------------+

```

### 2.1. ① タイトル画面 (Title Scene)
* **構成要素:**
  * ゲームタイトルロゴ（PixiJSテキスト、発光エフェクト推奨）
  * 「START」ボタン（タッチ認識領域）
  * ハイスコア表示（ローカルストレージから取得した過去の最高距離）
* **処理:**
  * 画面全体の静的な初期化。
  * STARTボタンがタップされたら、ゲームプレイ画面へ遷移。

### 2.2. ② ゲームプレイ画面 (Gameplay Scene)
* **構成要素:**
  * プレイヤー（宇宙船）、浮島（プラットフォーム）、アイテム（燃料タンク）、背景（星空等）、UI（スコア、燃料ゲージ）。
* **処理:**
  * 後述する物理演算、入力監視、ステージ生成、コリジョン判定のループ（`ticker`）を毎フレーム実行。
  * ゲームオーバー条件を満たした時、スコア表示画面へ遷移。

### 2.3. ③ スコア表示画面 (Score Scene)
* **構成要素:**
  * 今回のスコア（到達距離）
  * ハイスコア更新時の祝賀テキスト（更新時のみ表示）
  * 「RETRY」ボタン（即座にゲームプレイ画面へ遷移）
  * 「TITLE」ボタン（タイトル画面へ遷移）
* **処理:**
  * 今回のスコアをハイスコアと比較し、必要であればローカルストレージを更新。

## 3. コアメカニクスと物理挙動（最重要仕様）

### 3.1. 宇宙船の操作とジェット噴射方向
* **入力方式:** 画面の「タッチ（ホールド）およびドラッグ」。
* **角度の決定（相対位置方式）:** * ジェットの射出方向（ベクトル）は、**「宇宙船の中心座標」から「現在の画面タッチ（ドラッグ）座標」を結んだ直線方向**を指す。
* **角度の制限（クランプ）:**
  * ジェットの噴射口は、宇宙船から見て**真上（0度）から真下（180度）までの「左半円」**にのみ制限される。
  * タッチ位置が右側にある場合でも、角度の計算結果は0度（真上）または180度（真下）の限界値にクランプされる。
* **推進力（作用・反作用）:**
  * ジェットはタッチ位置（左半円内）に向けて噴射されるため、宇宙船が受ける推進力（加速度）は**タッチ位置とは「真逆の方向」**（右側・上側など）へ働く。
  * 結果として、プレイヤーは「右に進みたいときは宇宙船の左下をタッチする」「上に強く飛びたいときは宇宙船の真下をタッチする」という操作になる。

### 3.2. 物理演算アルゴリズム（毎フレーム処理）
毎フレーム、以下の数式・ロジックに基づき宇宙船の速度・位置を更新する。

1. **ジェットON時（タッチ中かつ燃料あり）:**
   * 宇宙船とタッチ位置の差分から角度 $\theta$ (ラジアン) を計算：
     $$\theta = \text{atan2}(y_{\text{touch}} - y_{\text{ship}}, x_{\text{touch}} - x_{\text{ship}})$$
   * $\theta$ を $0 \le \theta \le \pi$ の範囲にクランプする。
   * 加速度 $A_{\text{thrust}}$ を速度に加算（作用反作用のため減算処理）：
     $$v_x = v_x - \cos(\theta) \times A_{\text{thrust}}$$
     $$v_y = v_y - \sin(\theta) \times A_{\text{thrust}}$$
2. **環境特性（常に適用）:**
   * **重力:** $v_y = v_y + G$ （下方向への一定の加速度）
   * **空気抵抗（摩擦・慣性の維持）:** $$v_x = v_x \times \mu_x$$
     $$v_y = v_y \times \mu_y$$
     （※ $\mu_x, \mu_y$ は1未満の係数。これによりジェット停止時も滑らかに慣性移動しながら落下する）
3. **位置更新:**
   * $x_{\text{ship}} = x_{\text{ship}} + v_x$
   * $y_{\text{ship}} = y_{\text{ship}} + v_y$

### 3.3. 燃料（エネルギー）システム
* **消費:** タッチによるジェット噴射中、燃料（`fuel`）は一定レートで減少する。燃料が0になるとタッチしていてもジェットは噴射されない。
* **自動回復（スマートチャージ）:**
  * ジェットの噴射を停止（指を離す、または燃料切れ）すると、**「待機時間（Cooldown：1.0秒）」**のカウントダウンが始まる。
  * 待機時間中に再度タッチされた場合は、カウントダウンはリセットされ消費フェーズに戻る。
  * 待機時間が0に達すると、燃料が毎フレーム一定レートで自動回復し始め、現在の最大上限値（`maxFuel`）に達するまでチャージされる。
  * **重要:** 燃料は「空中を慣性飛行している最中」であっても、噴射を止めて条件を満たせば自動回復する（着地を必須としない）。

## 4. ステージ生成とカメラシステム

### 4.1. カメラ追従型スクロール
* 宇宙船はステージ内を右方向へと進む。
* 宇宙船の $x$ 座標が、画面横幅の40%の位置（デッドゾーン境界）を超えた場合、超えた量 $\Delta x$ だけ、ゲーム空間を保持するメインコンテナの $x$ 座標を減算（$- \Delta x$）する。これによりカメラが右に追従している表現を行う。
* 宇宙船が画面左端（カメラの左境界線）から見切れるような挙動（左への逆走）は、前述の物理制限（右方向のみへの加速）により構造的に発生しにくくするが、万が一見切れる場合は画面端でクリップする（ゲームオーバーにはしない）。

### 4.2. 浮島（プラットフォーム）の無限ランダム生成
* **形式:** 宙に浮いた様々な幅（`width`）と高さ（`y`座標）を持つ矩形のブロック。
* **生成アルゴリズム:**
  * カメラが右に進むのに合わせ、画面右外（見えない領域）に新しい浮島をランダムにトリガー生成する。
  * 浮島同士の間隔（谷の深さ・幅）は、ゲーム進行度（進んだ距離スコア）に応じて、プレイヤーが慣性ジャンプで届く限界を超えない範囲で徐々に広く、意地悪に調整される（難易度の上昇）。
* **破棄:** 画面左外（カメラの追従によって完全に見えなくなった領域）に外れた古い浮島オブジェクトは、メモリ解放のため即座に配列から削除（`destroy`）する。

### 4.3. 当たり判定（コリジョン）
* 宇宙船の底面と、浮島の上面のみに簡易的なAABB（軸平行境界ボックス）判定、または線分交差判定を行う。
* 宇宙船が「上から下へ落下している時」に浮島上面に衝突した場合のみ「着地」とみなし、$v_y = 0$ にして浮島の上に留まらせる。
* 燃料が0の状態で浮島に着地した場合、プレイヤーは動けなくなるが、自動回復の待機時間を経て燃料がチャージされれば、再び飛び立つことが可能。

## 5. アイテムシステム

### 5.1. 燃料タンク（上限アップアイテム）
* **配置:** 浮島の上に一定の確率でランダムに生成・配置される。
* **効果:** 宇宙船の当たり判定がアイテムに重なった時、以下の効果が即座に発動する。
  1. 現在の燃料値（`fuel`）を一定量、または全回復する。
  2. **燃料の「最大上限値（`maxFuel`）」を一定量プラス加算（育成・成長要素）**する。
* **目的:** 遠くへ行けば行くほど燃料タンクの恩恵で最大容量が増え、より長い谷を大ジャンプで越えられるようになる。

## 6. ビジュアル・グラフィック仕様（アセットレス）

すべての要素は外部画像を使わず、PixiJSの `PIXI.Graphics` でコード描画する。サイバーパンク/ネオン調の配色にすることでアセットレスでも見栄えを担保する。

| オブジェクト名 | 描画形状・スタイル | 配色イメージ |
| :--- | :--- | :--- |
| **宇宙船 (Ship)** | シャープな三角形、またはロケット型の多角形。後方にノズル。 | `#00e5ff` (シアン・ネオンブルー) |
| **浮島 (Platform)** | 上部に太いラインが入った角丸長方形。 | 本体 `#1a1a24` / 上面 `#39ff14` (ネオングリーン) |
| **アイテム (Item)** | 発光（Glow）しているように見える小さな円、またはひし形。 | `#ff007f` (ネオンピンク) またはイエロー |
| **背景 (Background)**| 遠景として、ランダムな位置に配置されたサイズの異なる微細な円（星）。 | `#ffffff` (輝度の強弱をつける) |

### 6.1. ジェットパーティクル演出
* **トリガー:** タッチ入力があり、燃料が減少しているフレーム。
* **生成ロジック:**
  * 宇宙船のノズル位置（座標）から、毎フレーム数個の小さな正方形または円（サイズ: 2px〜5px）のグラフィックオブジェクトを生成。
  * 排出される速度ベクトルは、**現在のジェット噴射角度 $\theta$ とは「真逆の方向」**（$\theta + \pi$）に対して、若干のランダムな分散（ノイズ）を加えた方向。
* **ライフサイクル:**
  * 生成されたパーティクルは、毎フレーム徐々にサイズが縮小（`scale`を減算）し、透明度（`alpha`）が下がる。
  * `alpha <= 0` になったパーティクルは、配列およびステージから完全に削除する。

## 7. 実装パラメーター（チューニング用初期値）

開発開始時の目安となる初期設定値。

```javascript
const GAME_PARAMS = {
    // 物理演算関連
    gravity: 0.25,          // 重力加速度 (G)
    frictionX: 0.98,        // X方向の空気抵抗（慣性維持率）
    frictionY: 0.99,        // Y方向の空気抵抗
    thrustPower: 0.4,       // ジェットの推進力
    
    // 燃料システム関連
    initialMaxFuel: 100,    // 初期燃料上限
    fuelDecreaseRate: 0.5,  // 1フレームあたりの燃料消費量
    fuelRecoverRate: 0.8,   // 1フレームあたりの燃料自動回復量
    cooldownDuration: 1.0,  // 再チャージが始まるまでの待機時間（秒）
    
    // アイテム効果
    fuelMaxUpgradeAmount: 15 // アイテム1個取得時の上限上昇値
};

```
